# Handoff AnimGIF — état au 2026-07-08 (fin de session)

## Le projet en une phrase

Générateur de GIF animés 100 % client-side (image → retouches → effets composables →
prévisualisation → export gifski-wasm), **v1 + v2 lot 1 livrés et en production** :
https://sandjab.github.io/animgif/ (repo GitHub `Sandjab/animgif`, public).

## État exact

- Branche `main`, tout poussé, arbre propre. **Chaque push sur main déploie en prod**
  (GitHub Actions : test + build + Pages) — ne pousser qu'un état vérifié.
- 45 tests Vitest verts (`npm test`), build propre (`npm run build` = tsc strict + vite).
- Docs de référence dans `docs/superpowers/` : specs v1 et v2, plans v1 et v2
  (code complet par tâche), `smoke-test.md` (15 points).

## Livré

**v1** : import (picker + drag&drop), retouches (luminosité/contraste/saturation,
flips, rotation 90°, couleur de fond), détourage IA local (imgly isnet_quint8
auto-hébergé dans `public/bg-removal/`, lazy), effets composables par matrices
affines (Ken Burns à rectangles éditables sur canvas, rotation, translation, bounce)
avec easings, lecteur (délai/FPS, boucles infinie / n fois / ping-pong), export
gifski-wasm en worker (dimensions verrouillables, qualité, garde-fou mémoire).

**v2 lot 1** : flip 3D (`spin3d`, projection parallèle en cos), 5 filtres natifs
(sépia, N&B, teinte, flou, inversion via `ctx.filter`), lecture inversée
(preview + export), décimation de frames à l'export (durée totale préservée),
« n fois » qui s'arrête vraiment en preview, export par tranches avec progression
déterminée au rendu + bouton Annuler (efficace pendant rendu ET encodage).

## Architecture (stable, ne pas casser)

- Moteur pur DOM-free et testé : `src/effects/` (easing, matrix, effects, timeline)
  + `computeFrameMatrices`. Un effet = `(t, params, view) → matrice affine` ;
  composition par multiplication ; kenBurns remplace le cadrage de base « contain ».
- `src/state.ts` : store pub/sub, fusion superficielle, ne jamais muter `get()`.
- UI vanilla TS : `src/ui/` (sourcePanel, canvasView avec hook `setOverlay`,
  animPanel avec re-render sur signature des kinds, kenBurnsEditor), `preview.ts`,
  `export.ts`. L'UI ne parle qu'au store.

## Pièges connus (durement acquis — vérifiés sur pièce)

1. **gifski-wasm `repeat`** : `<0` = boucle infinie, `Finite(0)` = UNE lecture sans
   extension NETSCAPE (pas infini !). Mapping : `count → loopCount-1`, sinon `-1`.
   Long commentaire dans `export.ts` — le préserver.
2. **Transfert des buffers vers le worker** : le ping-pong duplique des ImageData ;
   un ArrayBuffer ne se transfère qu'une fois → déduplication obligatoire dans
   `encodeInWorker` (régression DataCloneError déjà vécue et hotfixée).
3. **`@imgly/background-removal` épinglé à 1.7.0 exactement** : les assets de
   `public/bg-removal/` sont un snapshot de cette version (seul isnet_quint8 hébergé,
   les autres variantes de resources.json sont VOLONTAIREMENT absentes du disque).
4. Diagnostics LSP « Cannot find module » sur les fichiers de test = bruit périmé de
   la phase rouge TDD ; foi à `npx tsc --noEmit` et `npm test`.
5. ~24 Mo de wasm onnxruntime mort dans `dist/assets` : jamais téléchargé par les
   visiteurs (analysé), décision documentée de le laisser.
6. Décimation : opère sur l'ordre de BASE puis reconstruit le ping-pong (jamais sur
   l'ordre final — sinon doublons dos à dos) ; délai à l'échelle exacte de l'ordre final.
