## Livré cette session — lot 2a (export MP4)

Second format d'export **MP4/H.264** à côté du GIF, **sans changer l'architecture ni
régresser le chemin GIF**. Seul le backend d'encodage est neuf ; tout l'avant du pipeline
(ordre des frames reverse/ping-pong/décimation, rendu par tranches annulable, garde-fou
mémoire, progression) est réutilisé.

- **Encodage** : `src/mp4.ts` (`encodeMp4`, `isMp4Supported`) via **mediabunny**
  (`Output` + `CanvasSource` codec `avc` + `BufferTarget` + `Mp4OutputFormat`), sur le
  **thread principal** (WebCodecs asynchrone/non bloquant, backpressure gérée par la lib).
- **Helpers purs testés** : `src/mp4Params.ts` (`evenDim`, `qualityToBitrate`).
- **État** : `format: 'gif' | 'mp4'` (défaut gif) dans `src/state.ts`.
- **UI** (`index.html` + `src/export.ts`) : radios Format GIF/MP4, libellé de bouton
  dynamique, ligne d'aide « une seule passe ; boucle côté lecteur », progression
  **déterminée aux deux phases** en MP4, curseur qualité remappé en bitrate, annulation
  réutilisée.
- **Chargement paresseux** : mediabunny chargé par `import('./mp4')` uniquement à la 1re
  sélection MP4 → chunk séparé (~42 Ko gzip) hors bundle initial ; les visiteurs GIF-only
  ne le téléchargent jamais. Détection de support AVC paresseuse (`canEncode`).

## Pièges connus (durement acquis — vérifiés sur pièce)

Ceux du handoff précédent restent valides (sémantique `repeat` de gifski, déduplication
des buffers transférés au worker, imgly épinglé 1.7.0, bruit LSP « Cannot find module »,
wasm onnx mort, ordre de décimation). Nouveaux (lot 2a) :

7. **mediabunny 1.50.7 : la classe target est `BufferTarget`** (PAS `ArrayBufferTarget`,
   qui n'existe pas — la doc et d'anciens exemples le citent à tort). `target.buffer` est
   `ArrayBuffer | null` (`null` seulement avant `finalize()`). Codec H.264 = chaîne `'avc'`.
   Support via `await canEncode('avc')`.
8. **`await output.start()`** : le démarrage de l'`Output` est asynchrone — l'oublier crée
   une unhandled rejection silencieuse si le démarrage échoue (le code « marche » sinon par
   un détail interne non garanti).
9. **Erreurs d'encodage → `output.cancel()`** : la boucle `source.add`/`finalize()` est sous
   try/catch qui appelle `output.cancel()` avant de remonter — sinon fuite d'un encodeur
   WebCodecs (ressource matérielle limitée).
10. **H.264 exige des dimensions paires** : `evenDim` arrondit au pair AVANT le rendu (le
    MP4 sort donc en dims paires même si l'utilisateur saisit un impair).
11. **`URL.revokeObjectURL` différé** (`setTimeout`) dans `download()` : une révocation
    synchrone après `click()` fait échouer le téléchargement sur Firefox / certains Safari.
12. **Chargement paresseux du support MP4** : détection à la 1re sélection (pas au
    démarrage), enveloppée dans try/catch (échec réseau ou `VideoEncoder` absent en
    contexte non sécurisé → repli GIF sans planter l'UI, sans désactiver l'option car
    l'échec peut être transitoire). Garde de fraîcheur `if (!fmtMp4.checked) return;` après
    les `await` (course de double-sélection).
